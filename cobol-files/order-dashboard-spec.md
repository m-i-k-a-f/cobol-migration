# 株式注文フロー監視ダッシュボード 要件定義書

**プロジェクト名：** 東邦フィナンシャルシステムズ — リアルタイム注文監視システム（フロントエンド PoC）
**作成：** ワトソン株式会社 運用チーム SE
**ステータス：** PoC 設計中（実装担当：IBM Bob）

---

## 1. 背景・目的

東邦フィナンシャルシステムズでは、株式の注文処理を複数の COBOL バッチ・オンラインプログラムで管理しているが、**注文の処理状況をリアルタイムで把握できる画面が存在しない**。

運用チームから「今どれだけの注文が流れているか、異常な偏りがないか、ひと目で確認できるダッシュボードが欲しい」という要望が出ている。

本 PoC では、ダミーデータを使ってダッシュボードの **HTML 自己完結版（バックエンド不要）** を作成し、UI・指標の妥当性を検証する。

---

## 2. 表示する指標（KPI）

### 2-1. ヘッダーエリア（サマリー）

| 指標名 | 説明 | 更新頻度 |
|---|---|---|
| 本日の注文件数合計 | 買い＋売り＋取消の総件数 | 画面ロード時 |
| 約定件数 / 約定率 | 約定済み件数と（約定/総注文）% | 画面ロード時 |
| 未約定件数 | 受付済みで約定待ちの件数 | 画面ロード時 |
| 取消件数 | 取消処理された件数 | 画面ロード時 |

### 2-2. 注文フロー（メインエリア）

**注文ステータスのパイチャートまたはバーチャート**
- ステータス区分：`受付中` / `約定済` / `一部約定` / `取消済` / `エラー`
- 件数と割合（%）を両方表示

**銘柄別注文件数ランキング（上位10銘柄）**
- 銘柄コード・銘柄名・注文件数・売買区分（買い優勢 / 売り優勢）
- 横棒グラフで件数を可視化

### 2-3. 時系列エリア

**時間帯別注文件数（折れ線グラフ）**
- X軸：09:00〜15:30（30分刻み）
- Y軸：注文件数
- 買い・売り・取消を系列分け

**異常検知アラート欄**
- 直近1時間で注文件数が前時間帯比 +50% 超の場合に黄色ハイライト
- エラーステータスが5件超の場合に赤ハイライト

### 2-4. 注文明細テーブル（直近20件）

| カラム | 内容 |
|---|---|
| 注文ID | `ORD-XXXXXXXX` 形式 |
| 時刻 | HH:MM:SS |
| 銘柄コード | 4桁数字（例：7203） |
| 銘柄名 | 日本語（例：トヨタ自動車） |
| 売買区分 | 買 / 売 |
| 注文株数 | 数字（100株単位） |
| 注文単価 | 数字（円） |
| ステータス | 受付中 / 約定済 / 一部約定 / 取消済 / エラー |

## 3. UI・デザイン要件

- **カラーテーマ：** ダーク系（黒〜濃紺背景）。証券端末らしい緊張感のある配色
- **フォント：** モノスペースを数値に使用。日本語は読みやすいゴシック体
- **グラフライブラリ：** 外部 CDN 不可。SVG または Canvas で自前描画（純粋な自己完結HTML）
- **アニメーション：** 数値のカウントアップ演出（画面ロード時に 0 から最終値まで）
- **レスポンシブ：** 不要。横幅 1280px 固定想定（監視端末での表示）
- **言語：** 日本語

---

## 4. ダミーデータ仕様

HTML ファイル内の JavaScript に以下の条件でダミーデータを埋め込むこと。

- 銘柄：日本を代表する実在銘柄コード・銘柄名 10種（例：7203 トヨタ、6758 ソニー等）
- 注文件数：合計 **847件**（内訳は自然な比率で設計すること）
- 時間帯：09:00〜15:30 の市場時間内
- 異常検知トリガー：**13:00〜13:30 の時間帯で注文急増**（アラートが1件以上発火するように設定）
- エラー件数：**7件**（エラーアラートが発火する条件を満たすこと）

---

## 5. 技術制約

- 単一 HTML ファイルで完結すること（`.html` 1ファイルのみ、外部ファイル参照なし）
- CDN・外部 URL への参照禁止
- バックエンド・サーバー不要
- ブラウザで直接開いて動作すること
